## Segment Analytics Tracking

Analytics tracking enables historical analysis of sessions, custom events, and revenue for a segment. Without it, only real-time statistics are available.

### Enabling Analytics Tracking

In the segment's **Segment Details** section, toggle on **Analytics Tracking**.

- Maximum **25 segments** per app can have tracking enabled
- Prioritize segments tied to key campaign outcomes (sessions, revenue, purchases)

### Available Reports When Tracking is Enabled

| Report | Location | What You See |
|--------|----------|--------------|
| Revenue & Purchases | **Analytics > Revenue Report** | Revenue/purchases over time, filterable by segment |
| Sessions | **Home** page | Sessions over time for the segment |
| Custom Events | **Analytics > Custom Events Report** | Custom event counts over time |
| Query Builder | **Analytics > Query Builder** | Campaign/Canvas performance broken down by segment |

### Comparing Segments on Revenue Report

1. Go to **Analytics > Revenue Report**
2. Set **Breakdown** dropdown to **By Segment**
3. Select segments under **Breakdown values**
4. Toggle segment visibility by clicking segment names above the graph

Supports any custom time range.

### Query Builder Integration

With tracking enabled, use built-in report templates to break down performance metrics (campaigns, Canvas, variants, steps) by segment. Access via **Segment data** in the Query Builder.
