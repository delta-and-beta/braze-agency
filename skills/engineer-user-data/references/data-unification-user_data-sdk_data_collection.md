# SDK Data Collection

Braze SDK integration supports three tiers of data collection: **minimum** (non-configurable, required), **optional default** (auto-collected, can opt out), and **opt-in** (not collected by default).

## Minimum Integration Data

Always collected; non-configurable. Does **not** count toward data point usage — except session start/end.

| Attribute | Description |
|-----------|-------------|
| `App-Version-Name` / `App-Version-Code` | Most recent app version |
| `Country` | Via IP geolocation, device locale fallback, or `setCountry()`. Once set manually (SDK, REST API, or CSV), SDK stops auto-updating it. |
| `Device ID` | Randomly generated string per device |
| `OS` and `OS version` | Device/browser and version |
| `Session start` / `Session end` | Session boundaries (timing is developer-configurable per platform) |
| `SDK message interaction data` | Push opens, in-app message interactions, Content Card interactions |
| `SDK version` | Current SDK version |
| `Session ID` and `session timestamp` | Random string + timestamp; used for session continuity and message re-eligibility |

Session data drives engagement analytics and controls eligibility for in-app messages / Content Cards (synchronized on session start).

## Calculated Metrics (Server-Side)

Generated by Braze services from SDK data — not tracked by SDK directly, but visible on user profiles:

| Metric | Type |
|--------|------|
| First / Last used app | Time |
| Total session count | Number |
| Sessions in last Y days | Number + time |
| Last received any/email/push message | Time |
| Received message from campaign (with optional tag) | Boolean |
| Retarget campaign (opened/clicked email, push, IAM) | Boolean |
| Clicked card | Number |
| Uninstalled | Boolean + time |

## Tag Manager / CDP Warning

For **minimum integration only** with mParticle, Segment, Tealium, or GTM:
- **Mobile**: Must manually update code — mParticle/Segment cannot configure this through their platform
- **Web**: Braze must be integrated natively — tag managers cannot support minimum integration configuration

## Data Collection Tiers Summary

| Tier | Behavior | Configurable |
|------|----------|--------------|
| Minimum integration | Always collected | No |
| Optional (default on) | Auto-collected; opt-out available | Yes — disable per attribute |
| Optional (default off) | Not collected; opt-in required | Yes — enable per use case |
| Personalized | Custom attributes beyond defaults | Yes — developer-defined |
