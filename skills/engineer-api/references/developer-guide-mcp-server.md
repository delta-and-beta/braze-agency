### What is an MCP Server?

An MCP (Model Context Protocol) server exposes tools and resources that AI agents can invoke at runtime. In Nick's context, the MCP server provides **semantic search** over assembled plugin knowledge bases.

### Nick MCP Server

- **Start the server:** `nick mcp`
- **Generate config:** `nick mcp-setup`
- **Primary tool:** `semantic_search` — queries vector embeddings stored in `memory.db`
- **Storage backend:** SQLite with local vector embeddings
- **Default embedding model:** `Xenova/all-MiniLM-L6-v2` (384 dimensions)

### Integration with Claude Code

```json
// Example MCP config entry (generated by nick mcp-setup)
{
  "mcpServers": {
    "nick": {
      "command": "nick",
      "args": ["mcp"]
    }
  }
}
```

### Developer Notes

- The MCP server is optional — plugins function without it, but semantic routing degrades to keyword matching
- `memory.db` is generated during Step 11 (Memory) of the pipeline and stored at `~/.nick/plugins/{name}/memory.db`
- The server exposes a `plugin_path: "all"` option for federated search across all installed plugins
