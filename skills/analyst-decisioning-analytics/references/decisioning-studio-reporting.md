# Decisioning Reporting Overview

BrazeAI Decisioning Studio™ reporting provides performance metrics, data health monitoring, and system change tracking for AI-powered decisioning agents.

## Prerequisites

- Active contract for Braze + BrazeAI Decisioning Studio™
- Feature enabled by CSM
- At least one live Decisioning Studio agent

## Accessing Reports

Navigate to **AI Decisioning** > **BrazeAI Decisioning Studio™**, then select an agent.

## Date Range Controls

- Change date range via calendar dropdown after opening any report
- Set a **default start date** via Settings (<i class="fa-solid fa-gear"></i>)
- **Exclude specific dates** globally for an agent — excluded dates are filtered from all reports

## Available Reports

| Report | Purpose |
|--------|---------|
| **Performance** | High-level metrics comparing treatment vs. control groups; includes Trending and Driver Tree views |
| **Insights** | How recommendation options are generated — agent preferences and SHAPs reports |
| **Diagnostics** | Outbound/inbound data health: recommendation volume, data feed monitoring |
| **Timeline** | Visual log of agent runs, configuration changes, and guardrail updates alongside performance metrics |
