# Push Unit Tests (iOS Legacy SDK)

Unit tests to verify your app delegate correctly implements push notification integration. Passing tests indicate the code-based push setup is functional; failures may indicate a missed step or valid customization.

## Setup Steps

### 1. Create a Unit Testing Bundle

In Xcode: **File > New > Target** → "Unit Testing Bundle". Set "Target to be Tested" to your main app target. (Skip if one already exists.)

### 2. Add Braze SDK to Test Target

Using CocoaPods:

```ruby
target 'YourAppTarget' do
  pod 'Appboy-iOS-SDK'

  target 'YourAppTargetTests' do
    inherit! :search_paths
  end
end
```

### 3. Add OCMock to Test Target

Add [OCMock](https://ocmock.org/) via CocoaPods, Carthage, or static library.

```ruby
target 'YourAppTarget' do
  pod 'Appboy-iOS-SDK'

  target 'YourAppTargetTests' do
    inherit! :search_paths
    pod 'OCMock'
  end
end
```

### 4. Install Libraries

```bash
pod install
```

Open the `.xcworkspace` file generated by CocoaPods.

### 5. Add Push Test File

Create a new Objective-C file in your unit tests target. If prompted about an Objective-C bridging header (Swift project), click **Don't Create** — it's optional.

Copy the contents of [`AppboyPushUnitTests.m`](https://github.com/Appboy/appboy-ios-sdk/blob/master/HelloSwift/HelloSwiftTests/AppboyPushUnitTests.m) from the HelloSwift sample app into the new file.

### 6. Run Tests

Run the unit test suite (**Cmd+U** in Xcode). Use as a one-time verification or include in your CI pipeline to catch regressions.

## Notes

- Tests use Objective-C even in Swift projects (bridging header not required)
- OCMock is required for mocking push-related delegate calls
- Applies to legacy Objective-C iOS SDK only
